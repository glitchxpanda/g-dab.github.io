Minecraft

其实，本来Minecraft中mine 和 craft 分别是“采矿”和“制作”的意思，后来传入中国时人们把mine译为“我的”，慢慢的官方名字也就是我的世界了

Minecraft 中文wiki https://minecraft.fandom.com/zh/wiki/Minecraft_Wiki

mc版本：Java版和基岩版

java版：用Java编写

基岩版：用C++编写

基岩版是最初的携带版，可以跨设备上使用；Java 版只支持电脑平台

在MOD方面，Java版拥有官方的datapacks 和Frge、LiteLoader、Fabric等Mod API，Mod多样性比基岩版丰富得多。想用模组，就选择Java吧

不过基岩版用C++编写，运行的性能会比Java好

基岩版只支持Windows，对macOs 和 linux 不友好

Java Server 开服指导

官方指南 https://minecraft.fandom.com/wiki/Tutorials/Setting_up_a_server

这里使用Java版本进行开服是因为官方发布的开服软件里Java版已经很成熟，而基岩版开服软件还只是处于alpha版本https://minecraft.fandom.com/wiki/Bedrock_Dedicated_Server

mc服务器总介 https://minecraft.fandom.com/wiki/Server

Warning：想在自己电脑上开服的小盆友千万要小心，不然很可能会使自己的电脑被攻击

SDK(Software Development Kit) 中文意思是“软件开发工具包”

也就是把辅助开发某一类软件的相关文档、范例和工具SDK是一系列文件的组合写成一个包，它为软件的开发提供一个平台(它为软件开发使用各种API提供便利)。

JDK(Java Development Kit) Java开发工具包，也就是Java的一种SDK

这是Sun Microsystems针对Java开发员的产品。自从Java推出以来，JDK就是Java SDK的代名词了，如果要开发Java就必须有个JDK，所以Java和JDK其实是几乎等同的，那么之后谈及的每个Java版本其实就等价于一个JDK版本

Java的版本（表格）（代号很有意思）

JRE(Java Runtime Enviroment) Java的运行环境（里面就含有JVM）,安装JDK的时候包含JRE，这样写完程序就能用JRE来运行试试了，其中安装JRE是一般会有两个JRE，一个用来运行JDK里的工具（这些工具也是用Java写的），另一个给我们用来执行程序

JVM（Java Virtual Machine）java 虚拟机

provides a portable execution environment for Java-based applications

遵循着Write once, run anywhere宗旨，它帮助Java程序能在任何操作系统上运行，也起到优化Java程序的内存管理的作用

不过它和虚拟机不同，它不创建虚拟操作系统，我感觉和docker的容器更像

执行Java的方式

  1. Java 类加载器 Java class loader：用于加载Java程序的各种各样的类文件

  2. Java 执行引擎 execution engine： 加载完类后用于对类的执行

OpenJDK 与 OracleJDK

去官网转转: OpenJDK OracleJDK

Java由SUN公司(Sun Microsystems）发明之后，在2006年开源，此时的JDK即为OpenJDK

几乎现有的所有JDK都派生自OpenJDK（IBM J9，Azul Zulu，Azul Zing和Oracle JDK）

2009年Oracle收购SUN公司后，也成为JDK的一个维护方，他推出的就是OracleJDK。所以OpenJDK是开源的，Oracle JDK是官方不开源的

了解了这些，下面就来介绍Java版服务器的搭建

服务器的安装与配置

  1. 在官网安装了Java后（我使用的是Oracle Java）

先检查一下Java是不是最新版本

  1. 下载开服核心 (click here) (这是一个jar归档文件，以zip格式搭建，聚合了开服所需的所有的java类和数据)

    把核心放在要搭建服务器的文件夹里（用于存放所有的配置和世界文件），打开终端进入这个文件夹

  2. 开启服务器的命令是

    –nogui ：就是no gui（图形界面）（但似乎加上它仍然会跳出图形界面）

  3. 此时服务器应该会立刻关闭，会有提示

    在文件夹里会多出eula.txt 打开并改成eula=true

    eula是终端用户许可协议（End User Licence Agreement），改为true也就是同意他的软件许可

    elua.txt里就有mojang公司的用户协议

    此时再运行服务器就可以正常进行了

    不过对Minecraft 服务器来说最重要的是运行内存，所以加上-Xmx选项可以指定运行内存限制， 一般-Xmx4G就很绰绰有余了

    通过

    可以查看更多的开服选项

  4. 那么现在如果想以自己的电脑作为服务端，让其他人能连到自己创建的世界里

    你可能需要server.property里的online-mode改为false，否则只有经过身份验证的用户才能加入，而一般大多数人只是用离线模式登录

    打开自己的本地启动器来进入正在运行的服务器世界里，只需在-多人游戏-服务器地址中用localhost就可

    但如果其他主机想连进来，就需要你有一个公网ip了

    不过如果是局域网联机就不需要公网ip 你可以用ipconfig（windows）命令查看当前主机ip，然后其他主机要连接服务器需要输入的服务器地址就是 服务器ip：25565(默认开放的是25565端口，开放的端口在服务器配置文件server.property可以找到server-port)

    不过直接在自己的主机上运行服务器毕竟不太好，很可能会其他进程产生冲突，文件系统的储存也不独立，使用docker是一个更好一点 的选项，把mc服务器和自己的主机也分离开。

    编写一个脚本来自动启动mc不失为一个方便的选项 （官方tutorial上都有详细指导）
